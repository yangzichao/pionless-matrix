# pionless-agent

This repository is a cross-platform research agent package.

## Expected Layout

- `src/agents/` — canonical agent definitions (.md with codex frontmatter); single source of truth
- `src/skills/` — source skill files with `<!-- include: -->` markers
- `src/skills/includes/` — modular skill fragments shared across tiers
- `shared/` — expanded skills and non-generated assets (scripts, .mcp.json)
- `claude/` — GENERATED by build.sh (agents as .md, stripped of codex config)
- `codex/` — GENERATED by build.sh (agents as .toml, converted from .md)
- `plugins/pionless-agent/` — GENERATED committed repo plugin (serves both platforms)

## Development Flow

1. Edit agent definitions in `src/agents/` (system prompts, turn protocol, platform config)
2. Edit skill modules in `src/skills/includes/` (research rules, writing rules, templates)
3. Edit tier-specific parameters in `src/skills/{tier}/SKILL.md`
4. Run `bash build.sh` — this expands includes, generates platform agents, and assembles dist
5. Test `dist/claude-plugin` and `dist/codex-plugin` separately

Never hand-edit files in `platforms/claude-code/agents/`, `platforms/codex/agents/`, `shared/skills/`, or `plugins/pionless-agent/` — they are build outputs.

## Research Output Convention

All research output **must** be written to the `deep-research/` directory at the project root. Never write research reports to the repository root or any other location.

Filename format: `YYYY-MM-DD-HHMM-topic.md` where:
- `YYYY-MM-DD` is the current date
- `HHMM` is the current hour and minute (24h format, no separator)
- `topic` is a short lowercase slug derived from the research question (e.g. `ai-agent-frameworks`, `react-vs-vue`)

Workspace files follow the same prefix: `YYYY-MM-DD-HHMM-topic.workspace.md`

Examples:
- `deep-research/2026-03-30-1423-ai-agent-frameworks.md`
- `deep-research/2026-03-30-1423-ai-agent-frameworks.workspace.md`
