# Teaching patterns

## 1. Controlled-depth loop

Use this loop repeatedly:
1. orient the learner
2. probe one branch
3. stop early enough
4. summarize locally
5. zoom back out
6. choose next branch

## 2. When to use bfs first

Start with breadth first when:
- multiple sibling concepts are in play
- the learner keeps confusing terms
- the topic is new and needs a map
- the user asked for comparison or overview

## 3. When to use dfs first

Start with depth first when:
- the learner asks about one specific bottleneck
- one local misunderstanding blocks everything else
- the user already has the map and wants refinement

## 4. Practical heuristics

- spend fewer turns on classification and more on teaching
- prefer one clean question over three nested ones
- once a concept is good enough for current goals, move on
- revisit branches later instead of solving everything in one pass
